Tabcast Broadcast and receive tab URL streams

Defining Tabcast groups

Below you can define the Tabcast groups you want to track and/or broadcast URLs to. For more information on using Tabcast in general, visit tabcast.net. Each Tabcast group is specified with the following details:

Nickname
A short memorable name for a group on a Tabcast server. It is just a local name, for your convenience. Group nicknames will be shown in Chrome's context menu.
Server URL
The URL of the Tabcast server for the group. If in doubt, use http://tabcast.net/ as your server. You can also run your own server (Get the Tabcast source from Github).
Group name
The name of the group on the Tabcast server. You can join an existing group or create your own.
Group password (Optional)
The password (if any) for the group. Groups without passwords can be accessed by anyone. Note that if you do not set a password on a new group and someone else does, you will then be locked out (unless they tell you the password).
User name (Optional)
The user name you wish to use on the server, if any. If you do not specify a username, your URLs will be shown as being from <anon>
Password (Optional)
A password to identify yourself on the server. You do not have to use a password. If you do not, other users can use the same username. In that case, if any of them does set a password for the username your use of it will fail thereafter (unless they tell you the password). Note: passwords are stored locally by Chrome, in plaintext, and are therefore not very secure. Do not re-use a password that you use elsewhere.
